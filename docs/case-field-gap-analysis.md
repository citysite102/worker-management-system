## 二、設計策略：以易用性為核心的重構方案

### 核心設計原則
1. **選雇主後自動帶入**：雇主相關欄位（電話、地址、被照顧者資料）選擇後自動填充，唯讀顯示，不需手動輸入
2. **選移工後自動帶入**：移工相關欄位（姓名、證件、體檢資料）選擇後自動填充，唯讀顯示
3. **流程化 Tab 設計**：將 Ragic 的長表單拆分為有意義的流程階段，每個 Tab 代表一個業務階段
4. **負責人快速指派**：流程步驟中的負責人欄位統一使用下拉選擇，不需打字

### 建議的 Tab 架構（重構 CaseDetail）

```
Tab 1: 基本資料
  - 案件編號（自動產生）、案件名稱、案件情況、負責人
  - 選擇雇主（下拉）→ 自動帶入：電話、地址、被照顧者姓名/資格
  - 選擇移工（下拉）→ 自動帶入：中英文姓名、證件號碼、有效日期、手機
  - 招募許可函（附件）
  - 需檢查標記、備註

Tab 2: 聘僱資料（原「資格」Tab 合併重組）
  - 本案件接續聘僱日期、期間長度、終止聘僱日期
  - 招募函代辦事項、聘僱函代辦事項
  - 聘僱後尚未完成保險
  - 勞動部聘僱許可函：收文號、類別、申請書送件日、發文日期、核准收件日
  - 聘僱許可函（附件）、聘僱情況、終止函（附件）

Tab 3: 通報/移民署
  - 通報書序號、入國通報申請日、證明書序號
  - 一站式類別、一站式序號、居留證申請送審日

Tab 4: 體檢管理（新增）
  - 前次體檢日期（從移工自動帶入，唯讀）
  - 體檢基準日
  - 入境3日體檢 + 報告附件
  - 6個月體檢 + 報告附件
  - 18個月體檢 + 報告附件
  - 30個月體檢 + 報告附件

Tab 5: 保險
  - 健保投保
  - 意外險 + 保單附件

Tab 6: 流程追蹤（新增，取代現有「媒合」Tab 的部分功能）
  選工媒合：國外仲介互貿、提供履歷、面試選工、確認工人、勞動條件確定
  文件處理：求才、初招函、工資切結書、簽證申請、入國通知單
  入境安排：勞工局入國通報、一站式、移民署居留證、勞動部聘僱許可函、入境3日體檢、薪資系統導入、健保申請、意外險投保
  在職管理：6個月體檢、1年居留證展延、18個月體檢、2年居留證展延、30個月體檢、護照展延、變更居住地點
  評鑑管理：委任服務契約書、服務紀錄表、非勞工行政服務、意外事件處理1955
```

### Phase 3（中優先，體檢追蹤）
